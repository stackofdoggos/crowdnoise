Current `split_stems` behavior:

- Uses Demucs model `htdemucs_6s` (6-stem separation).
- Writes split outputs as MP3 files under `output/htdemucs_6s/<song title>/`.
- Skips `.gitkeep` in `resources/` so Demucs only receives real audio files.
- Skips tracks that are already split (detects existing stem audio files in the output folder).
- Generates `songdetails.json` for each split song in its output directory.

`songdetails.json` format:

```json
{
  "schema_version": 1,
  "song_id": "kanye-west-homecoming",
  "title": "Kanye West - Homecoming",
  "sr": 44100,
  "song_length_ms": 32000,
  "instruments": {
    "bass": { "original_path": "output/htdemucs_6s/Kanye West - Homecoming/bass.mp3" },
    "drums": { "original_path": "output/htdemucs_6s/Kanye West - Homecoming/drums.mp3" },
    "other": { "original_path": "output/htdemucs_6s/Kanye West - Homecoming/other.mp3" },
    "vocals": { "original_path": "output/htdemucs_6s/Kanye West - Homecoming/vocals.mp3" }
  }
}
```

Current `isolate_drums` behavior:

- Python requirement is now `3.10+` (not hard-locked to `3.10`).
- Reads a drum stem input (for example `output/htdemucs_6s/<song>/drums.mp3`) and writes outputs in Demucs-style subfolders:
  - `output/trackDecomp/<song>/kick.mp3`
  - `output/trackDecomp/<song>/hats.mp3`
- Supports one-shot and hit export with `--export-hits`, writing:
  - `kick_times.csv`, `hats_times.csv`
  - `kick_one_shot.mp3`, `hats_one_shot.mp3`

Current `basic_pitch_to_midi` behavior:

- Python requirement is now `3.10+` (3.11 runs are supported).
- Shebang and CLI guidance were updated to run with `python3`.

Current `visualize_hit_times` behavior:

- Input: `kick_times.csv`, `hats_times.csv`, and a reference audio file (usually Demucs `drums.mp3`).
- Output: waveform plot with kick/hats hit markers overlaid (can display interactively or save PNG).
- Default visualization window is first 15 seconds (or full audio if shorter).
- Example output plots:
  - `output/plots/homecoming_hits.png`
  - `output/plots/drumtrack_hits.png`
